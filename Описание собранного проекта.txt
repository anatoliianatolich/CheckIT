Всем привет!
По ссылке https://github.com/vlasnet/CheckIT репозиторий с нашим проектом.
Проект собрал на базе create-react-app с установленными и настроенными eslint, prettier, css modules.
В качестве базы данных сейчас подключен firebase, авторизация пользователей тоже осуществляется через firebase, поэтому сейчас в проекте используются формы регистрации и логина с другого проекта, где я принимал участие. Когда будут готовы наши компоненты регистрации - заменим их.
Также в проекте прикручены болванки основной страницы с сайдбаром, и страницы профайла. Сделана и работает навигационная панель с минимально необходимой навигацией по сайту.

Для разворачивания и запуска проекта:
1. делаете клон удаленного репозитория
2. checkout на ветку develop и pull данных с удаленного репо.
3. переходите в папку client и запускаете инициализацию зависимостей проекта ( npm i )
4. по окончанию закачивания и установки зависимостей можно запускать проект ( npm start )
Когда проект запустится пройдите регистрацию (e-mail и пароль можно вводить любой, главное их помнить для повторных входов)

По дальнейшей работе с git-ом - предлагаю строить нашу работу по принципу, изложенному в статье, которую я раньше кидал (https://m.habr.com/post/106912/).
Соответственно, перед началом работы над своей частью кода необходимо спулиться и отбранчеваться от ветки develop, по окончанию работы подготовленный коммит подается в виде pull request в ветку develop. 
В случае возникновения конфликтов, их необходимо решить до пул реквеста. Если все будет ОК, то пул реквесты будут смержены в ветку develop и в ней будет находиться рабочая версия проекта со всеми последними принятыми обновлениями.
В ветку master будем заливать готовые рабочие версии проекта по окончанию спринтов.
